Java正则表达式

发表于 2020-04-12 | 分类于 Java
字数统计: 1.3k | 阅读时长 ≈ 4
正则表达式字符作用相关整理

相关类

Java正则表达式的典型使用为：

Pattern类是不可变类，可供多个并发线程使用
常用方法有：

修饰符或类型 方法 描述
static Pattern compile​(String regex) 将正则表达式字符串编译为Pattern对象
Matcher matcher​(CharSequence input) 创建一个匹配器根据编译好的Pattern对象来匹配给定的输入（注1）
static boolean matches​(String regex, CharSequence input) 编译给定的正则表达式，并将其与输入的字符序列匹配

注1：CharSequence是一个接口，代表一个字符序列，其中CharBuffer，String，StringBuffer，StringBuilder都是其实现类

Matcher类利用Pattern来对字符序列进行匹配操作
常用方法有：

修饰符或类型 方法 描述
boolean find() 尝试查找与Pattern匹配的输入序列的下一个子序列
boolean lookingAt() 返回目标字符串前面部分与Pattern是否匹配
boolean matches() 返回整个目标字符串与Pattern是否匹配
String group() 返回上一个与Pattern匹配的子串
int start() 返回上一个与Pattern匹配的子串在目标字符串中的开始位置
int end() 返回上一个与Pattern匹配的子串在目标字符串中的结束位置
Matcher reset() 重置匹配器

字符规则

合法字符

合法字符 说明
x 字符x（任何合法字符）
\0mnn 八进制数0mnn所表示的字符
\xhh 十六进制值0xhh所表示的字符
\uhh 十六进制值0xhh所表示的Unicode字符
\N{name} Unicode字符名称为”name”的字符
\t 制表符(‘\u0009’)
\n 换行符(‘\u000A’)
\r 回车符(‘\u000D’)
\f 换页字符(‘\u000C’)
\a 警报（bell）字符(‘\u0007’)
\e Escape字符(‘\u001B’)
\cx x对应的控制符，x必须为A~Z或者a~z（\cV匹配Ctrl+V）

特殊字符

特殊字符 说明
$ 匹配一行结尾（注2）
^ 匹配一行开头
() 标记子表达式开始和结束位置
[] 用于确定中括号表达式的出现频度
{} 用于标记前面子表达式的出现频度
* 指定前面子表达式可以出现零次或多次
+ 指定前面子表达式可以出现一次或多次
? 指定前面子表达式可以出现零次或一次
. 匹配除了换行符\n以外的任何单字符
\ 转义下一个字符，或指定八进制，十六进制字符
| 指定两项之间任选一项

注2：若要匹配字符本身，使用转义字符\，例如\*匹配*，而在Java字符串中，反斜杠本身需要编译，所以实际编程中需要\\*来匹配*

预定义字符

预定义字符 说明
. 可以匹配任何字符
\d 匹配0~9的所有数字
\D 匹配非数字
\s 匹配所有的空白字符，包括空格、制表符、回车符、换页符、换行符等
\S 匹配所有的非空白字符
\w 匹配所有的单词字符，包括0~9所有数字，26个英文字母和下划线(_)
\W 匹配所有的非单词字符

方括号表达式

方括号表达式 说明
表示枚举 [abc]表示a,b,c中任意一个字符
表示范围 [a-z]表示a~z范围内任意字符（注3）
表示求否 [^abc]表示非a,b,c的任意字符
表示与运算 [b-z&&[abc]]表示b~z与a，b，c的交集，即为b,c
表示并运算 与枚举类似[abc[bcd]]=[abcd]

注3：范围可以与枚举结合使用，如[a-ce-f]=[abcef]，同时可以利用此表达式匹配Unicode的中文字符

边界匹配符

边界匹配符 说明
$ 行的结尾
^ 行的开头
\b 单词的边界
\B 非单词的结尾
\A 输入的开头
\G 前一个匹配的结尾
\Z 输入的结尾，仅用于最后的结束符
\z 输入的结尾

数量表示符

数量表示符模式

  • 贪婪模式（Greedy）：数量标识符默认采用贪婪模式。贪婪模式会一直匹配下去，直到无法匹配为止。
  • 勉强模式（Reluctant）：用?后缀表示，只会匹配最少的字符。也称为最小匹配模式
  • 占有模式（Possessive）：用+后缀表示
贪婪模式 勉强模式 占用模式 说明
X? X?? X?+ X表达式出现零次或一次
X* X*? X*+ X表达式出现零次或多次
X+? X+? X++ X表达式出现一次或多次
X{n} X{n}? X{n}+ X表达式出现n次
X{n,} X{n,}? X{n,}+ X表达式至少出现n次
X{n,m} X{n,m}? X{n,m}+ X表达式至少出现n次，最多出现m次
